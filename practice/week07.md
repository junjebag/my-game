## 수업에서 배운 내용 적용

- `convert_alpha()`로 PNG 투명 배경 처리
- `pygame.transform.scale()`로 이미지 크기를 게임 규격(40x40 등)에 맞게 조절
- `os.path.join()`으로 경로를 처리해 Windows/Mac/Linux 어디서든 실행되게 함
- `pygame.mixer`로 효과음과 배경음악 재생, 배경음악은 `-1`로 무한 반복
- 수업 권장 폴더 구조 준수

## 추가로 연구한 내용

**Fallback 시스템**: 에셋 파일이 없어도 도형으로 대체 렌더링되게 해서 파일 누락 시 에러 없이 실행되도록 했다.

**오디오 지연 해결**: 피격 효과음이 늦게 나오는 문제를 발견해서, mixer 버퍼 크기를 기본값 4096에서 512로 줄여 해결했다.

**배경 이미지 전처리**: 실제 화장실 사진을 그대로 쓰면 배경이 너무 선명해서 게임 요소가 묻히는 문제가 있었다. PIL로 가우시안 블러와 밝기 조정을 적용해 화장실 느낌은 유지하면서 가독성을 확보했다.

**`BLEND_RGB_MULT` 블렌드 적용**: 시간 정지 시 적을 어둡게 표현할 때 사각형 오버레이 대신 곱셈 블렌드를 써서 스프라이트 모양을 유지하면서 어둡게 처리했다.

## AI(Claude) 활용 내역

### 사용한 주요 프롬프트 (요약)

1. **기획 단계**: "Superhot 시스템과 똥피하기를 합치면 어떤 게임이 나올까? 추가할 수 있는 요소 추천해줘."
2. **구현 단계**: "시간 정지 시스템 구현해줘. 플레이어가 움직이지 않으면 time_scale이 0으로 떨어지고, 움직이면 다시 올라오게."
3. **디버깅**: "스프라이트 주변에 사각형 잔상이 생겨. 원인이 뭐야?"

### AI 답변에서 도움이 된 것

- 시간 정지 시스템에서 `time_scale` 변수 하나로 모든 오브젝트 속도를 제어하는 구조가 명확해서 바로 적용할 수 있었다.
- fallback 렌더링 패턴(`if IMG_PLAYER is not None: blit else: draw.rect`)을 제안받아 에셋 없이도 게임이 돌아가게 만들었다.

### AI 답변을 수정하거나 버린 것

- 처음에 글리치 적의 이동 패턴을 너무 복잡하게 짜줘서 단순한 지그재그로 직접 수정했다.
- 무적 시간을 3초로 설정해줬는데 너무 길어서 1.5초로 줄였다.
- 디자인 감각과 게임성 판단(블러 강도, 밝기 값 등)은 직접 조정하며 결정했다.
