# 6주차 실습 기록

## 선택한 템플릿 및 주제
- 템플릿: Dodger (피하기 게임)
- 적용 기법: 화면 흔들기 (Screen Shake), 무적 프레임 (Invincibility Frames)
- 선택 이유: 낙하물에 맞는 순간 화면이 흔들리면 타격감이 생기고, 무적 프레임이 없으면 낙하물이 연속으로 맞을 때 목숨이 한꺼번에 다 깎여서 게임이 너무 어려워지기 때문에 선택했다.

## 사용한 AI 프롬프트 (요약)
1. 코드 구조 설명 요청
   - "아래는 Dodger 게임 코드야. 전체 구조를 설명해줘. 특히 낙하 오브젝트가 어떻게 생성되고 이동하는지, 플레이어와 충돌은 어떻게 처리되는지 알려줘."
2. 화면 흔들기 구현 요청
   - "내 Dodger 게임 코드에 screen shake를 추가해줘. 낙하물에 맞는 순간 화면이 흔들리게 해줘. 오프스크린 버퍼 방식으로 구현해줘."
3. 무적 프레임 구현 요청
   - "피격 직후 2초 동안 무적이 되게 해줘. 무적 중에는 캐릭터가 깜빡이게 표현해줘."

## AI 답변에서 도움이 된 것
- 화면 흔들기에서 `trigger_shake(8)` 한 줄만 추가하면 피격 시 자동으로 흔들리는 구조가 이해하기 쉬웠다.
- 무적 프레임에서 `invincible_timer // 4 % 2 == 0` 조건으로 깜빡임을 표현하는 방식이 간단해서 바로 적용할 수 있었다.
- 충돌 처리 부분에 `if collision and not invincible:` 조건만 추가하면 된다는 것을 알게 되었다.

## AI 답변을 수정하거나 버린 것
- 처음에 화면 흔들기를 카메라 좌표 오프셋 방식으로 구현해줬는데, 코드가 복잡해서 오프스크린 버퍼 방식으로 다시 요청했다.
- 무적 시간을 처음에 3초(180프레임)로 설정해줬는데, 너무 길어서 2초(120프레임)로 수정했다.
